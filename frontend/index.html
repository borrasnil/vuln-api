Broken Object Level Authorization (BOLA/IDOR)

APIs expose endpoints that handle object identifiers without verifying ownership. Any user can access anyone else's data by changing the ID.

Test Endpoints

Why It's Vulnerable

  • • No authentication required
  • • No ownership check on resource access
  • • Trusting user-provided IDs

Attack Example

curl http://localhost:5000/api/v1/users/2

Broken Authentication

Flawed login mechanisms allowing account takeover, brute force, and session manipulation.

Test Endpoints

Why It's Vulnerable

  • • No rate limiting
  • • No account lockout
  • • No verification on password reset

Attack Example

Try: admin / admin123

Broken Object Property Level Authorization

Excessive data exposure + mass assignment - returns all fields including sensitive data.

Test Endpoints

Why It's Vulnerable

  • • Returns all fields including password
  • • Allows modifying role, is_admin
  • • No field allowlist

Try fetching users - password exposed!
